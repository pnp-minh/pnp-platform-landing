# Demo Flow Design - Psychology & Structure

**Current Focus**: Version A (Revenue Growth) - Version B (Knowledge Retention) scheduled for next week

**Product Name**: Primer (formerly JAY - all references updated)

## Core Principles for Both Versions

### Psychology Rules:

1. **Instant gratification**: Wow moment in <60 seconds
2. **User agency**: User DOES something (not just watches)
3. **Progressive disclosure**: Don't show everything at once
4. **Emotional peak-end**: Best moment at middle, strong close
5. **Social proof injection**: "X agencies are testing this" somewhere visible

### UX Rules:

1. **No login required** for demo
2. **Pre-filled data** for speed (but user can edit)
3. **Live feedback**: Every action shows immediate result
4. **Clear progress**: "Step 1 of 3" or progress bar
5. **Embedded CTA**: Waitlist signup appears naturally, not forced

- **When**: After landing page CTA, before demo starts
